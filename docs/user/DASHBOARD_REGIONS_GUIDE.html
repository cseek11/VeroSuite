Part 1: Quick Start

Getting Started in 5 Minutes

What are Regions?

Regions are organized containers in your dashboard that hold widgets and content. Unlike the previous card system, regions are arranged in a structured grid layout that you can customize. Each region can be:

  • Moved anywhere on the grid
  • Resized to fit your needs
  • Collapsed to save space
  • Locked to prevent changes
  • Customized with colors and titles
  • Shared with specific users, roles, or teams
Key Concepts: Regions snap to a grid system for clean organization. All changes save automatically after 500ms. Real-time collaboration lets you see who else is viewing or editing.

Your First Dashboard

Step 1: Navigate to Dashboard

  1. Click on Dashboard in the main navigation
  2. If this is your first time, you'll see an empty dashboard with a message: "No regions configured"

Step 2: Add Your First Region

  1. Click the purple Settings button (‚öôÔ∏è) in the top-left corner of the screen
  2. In the settings panel that appears, click "Add Region"
  3. A dialog will appear showing available region types
  4. Select a region type (e.g., "Scheduling", "Reports", "Customer Search")
  5. The region will appear in your dashboard grid at the next available position
The region has resize handles (purple squares) on corners and edges. A drag handle (‚ãÆ‚ãÆ) appears in the top-left when you hover over the header.

Step 3: Customize Your Region

  1. Click the Settings icon (‚öôÔ∏è) in the region header
  2. In the settings dialog:
    • Change the Title (up to 100 characters)
    • Select Region Type from the dropdown
    • Customize Background Color, Header Color, and Border Color
  3. Click "Save" to apply changes

Step 4: Arrange Your Layout

  • Move a region: Click and drag the region header (‚ãÆ‚ãÆ icon) to a new position
  • Resize a region: Drag any of the purple resize handles on the corners or edges
  • Collapse a region: Click the collapse button (‚ñº) in the region header
  • Lock a region: Click the lock button (üîí) to prevent accidental changes

Essential Actions

Adding Regions

Method 1: Dashboard Settings Panel

  1. Click the purple Settings button (‚öôÔ∏è) in top-left
  2. Click "Add Region"
  3. Select a region type

Method 2: Keyboard Shortcut

  • Press Ctrl+K (or Cmd+K on Mac) to open Command Palette
  • Type "add region" and select the command
  • Choose region type

Method 3: Quick Number Keys

  • Press number keys 1-9 to quickly add common region types (when not typing in input fields)

Moving Regions

Drag and Drop:

  1. Hover over a region header
  2. Click and hold the drag handle (‚ãÆ‚ãÆ icon) that appears
  3. Drag to the desired grid position
  4. Release to drop
What happens: Other regions automatically shift to make room. The grid layout prevents overlapping. Changes save automatically.

Resizing Regions

  1. Hover over a region to see resize handles
  2. Corner handles (purple circles): Resize both width and height
  3. Edge handles (purple rectangles): Resize width (left/right) or height (top/bottom)
  4. Drag the handle to the desired size
  5. Release to apply
Constraints: Regions have minimum size limits. The grid enforces consistent spacing. Very large regions may require scrolling the canvas.

Deleting Regions

  1. Click the More Options button (‚ãÆ) in the region header
  2. Select "Delete" from the context menu
  3. Confirm the deletion
Note: Regions are soft-deleted and can be restored from version history.

Part 2: Core Features

Adding Regions

Available Region Types

When adding a region, you can choose from:

  • Scheduling - Calendar and schedule management
  • Reports - Report generation and viewing
  • Customer Search - Search and filter customers
  • Settings - Configuration options
  • Quick Actions - Common task shortcuts
  • Analytics - Data visualization and metrics
  • Team Overview - Team performance and status
  • Financial Summary - Financial data and summaries
  • Custom - Custom widget configuration

Resizing Regions

Resize Handles

Regions have 8 resize handles:

  • 4 corner handles (16px √ó 16px purple circles): Resize both dimensions
  • 4 edge handles (rectangular purple bars): Resize one dimension
    • Top/Bottom edges: Resize height
    • Left/Right edges: Resize width

Resize Behavior

  • Minimum Size: Each region has a minimum width and height
  • Grid Snapping: Resizing snaps to grid increments
  • Visual Feedback: The handle highlights on hover (opacity 0.8)
  • Smooth Animation: Resize operations are animated

Moving/Reordering Regions

Drag and Drop

  1. Hover over a region header to reveal the drag handle
  2. Click and hold the drag handle (‚ãÆ‚ãÆ icon)
  3. Drag to the desired grid position
  4. Release to drop
Visual Feedback: The region becomes semi-transparent while dragging. A preview shows where it will be placed. Other regions shift automatically.

Swap Behavior

  • If you drag a region onto another region's position, they swap positions
  • This is enabled by default
  • Prevents accidental overlapping

Collapsing/Minimizing Regions

Collapse a Region

  1. Click the collapse button (‚ñº) in the region header
  2. The region collapses to show only its header
  3. The collapsed region is moved off-screen (to column 1000)
  4. A minimized version appears in the Minimized Region Dock (top-right corner)

Minimized Region Dock

Location: Fixed position at top-right of screen

  • Shows all collapsed regions as compact cards
  • Each card displays the region type name
  • Click the restore button (‚õ∂) to restore a region
  • Horizontal scrolling if many regions are minimized
  • Auto-hides when no regions are minimized

Locking Regions

Lock a Region

  1. Click the lock button (üîí) in the region header
  2. The button changes to a locked state (üîí)
  3. The region border changes to red
  4. The region background becomes semi-transparent red

Locked Region Behavior

  • Cannot be moved - Drag handle is disabled
  • Cannot be resized - Resize handles are hidden
  • Cannot be deleted - Delete option is disabled
  • Can be viewed - Content remains visible
  • Can be unlocked - Click lock button again

Duplicating Regions

Duplicate a Region

  1. Click the More Options button (‚ãÆ) in the region header
  2. Select "Duplicate" from the context menu
  3. A new region is created with:
    • Same region type
    • Same configuration
    • Offset position (row+1, col+1)
Visual Feedback: Button shows "Duplicating..." while processing. Success toast notification appears. New region appears at offset position.

Opening Settings

Part 3: Advanced Features

Dashboard FAB (Floating Action Button)

Location: Top-left corner

Opening the Settings Panel

  1. Click the purple Settings button
  2. Settings panel slides in from the left
  3. Backdrop overlay appears (10% black opacity)

Settings Panel Contents

  • Add Region Button - Opens region type selector
  • Collaboration Status - Shows connection status and active collaborators
  • Undo/Redo Controls - Revert or reapply actions
  • Export/Import Layout - Download or upload layout JSON files
  • Zoom Controls - Zoom in, out, or reset view
  • Reset All Button - Resets zoom/pan and loads default layout
  • Fullscreen Toggle - Enter or exit fullscreen mode

Zoom and Pan

Zoom Controls

  • Zoom In: Click "+" button in Dashboard FAB (maximum: 200%)
  • Zoom Out: Click "-" button in Dashboard FAB (minimum: 25%)
  • Reset Zoom: Click "Reset" (‚Üª) button - returns to 100%

Pan Controls

Mouse Panning:

  1. Click and hold anywhere on the canvas (not on a region)
  2. Drag to pan the view
  3. Release to stop panning
Cursor changes to "grabbing" hand. Canvas moves smoothly. Regions maintain their positions.

Versioning System

Creating Versions

When to Create Versions:

  • Before making major layout changes
  • After completing a set of changes
  • Before sharing with team
  • As backup points

Create a Version

  1. Make your dashboard changes
  2. Open Version History (from Dashboard FAB or Command Palette)
  3. Click "Create Version"
  4. Fill in version details:
    • Status: Draft, Preview, or Published
    • Notes: Optional description
  5. Click "Create"

Version Statuses

  • Draft - Work in progress, only visible to you
  • Preview - Ready for review, visible to team members
  • Published - Active version, visible to all users with access

Publishing Versions

  1. Open Version History
  2. Find the version you want to publish
  3. Click "Publish" button
  4. Add publish notes (optional)
  5. Confirm publication
What happens: Previous published version becomes "Archived". New version becomes active. All users see the new layout.

Collaboration Features

Real-Time Presence

  • Viewing Indicators: User avatars appear on regions being viewed
  • Editing Indicators: Yellow highlight on regions being edited
  • Connection Status: Green WiFi icon = connected, Red = disconnected

Region Locking (Collaboration)

How it works:

  • When you start editing a region, a soft lock is acquired
  • Other users see the lock indicator
  • Lock expires after 5 minutes of inactivity
  • Automatically released when you finish editing

Conflict Resolution

Editing Conflicts:

  • If someone else is editing, you'll see a conflict alert
  • Options: Wait for them to finish, Request to take over, Work on a different region
  • Locks expire after 5 minutes automatically

Undo/Redo System

How It Works

  • Tracks up to 50 state changes
  • Saves state after each action
  • Debounced (500ms) to avoid excessive saves
  • Automatically clears old history when limit reached

Using Undo/Redo

Undo: Press Ctrl+Z (Windows/Linux) or Cmd+Z (Mac)

Redo: Press Ctrl+Shift+Z (Windows/Linux) or Cmd+Shift+Z (Mac)

What Can Be Undone: Add/delete/move/resize regions, change settings, collapse/expand, lock/unlock. Version operations and import/export cannot be undone.

Export/Import Layouts

Export Your Dashboard

  1. Open Dashboard FAB (top-left)
  2. In settings panel, find "Layout" section
  3. Click "Export" button
  4. JSON file downloads automatically
  5. File named: dashboard-layout-YYYY-MM-DD.json

Import a Dashboard

  1. Open Dashboard FAB
  2. In settings panel, find "Layout" section
  3. Click "Import" button
  4. File picker opens
  5. Select JSON file (`.json` extension)
  6. File is validated and imported
Best Practices: Export current layout before importing. Validate JSON file before importing. Test import in preview environment first. Keep backup of working layouts.

Part 6: Troubleshooting & Best Practices

Common Issues

Region Not Loading

Symptoms: Region appears but content is blank, loading spinner never completes, error message displayed.

Solutions:

  1. Check Internet Connection - Verify network connectivity
  2. Refresh the Page - Press F5 or Ctrl+R
  3. Check Browser Console - Open Developer Tools (F12) and check for errors
  4. Check Region Configuration - Verify widget type is set correctly
  5. Check Permissions - Verify you have read access to the region

Can't Edit Region

Symptoms: Buttons are disabled, can't move or resize, settings dialog won't open.

Solutions:

  1. Check Lock Status - Look for lock icon (üîí) in header
  2. Check Permissions - Verify you have edit permissions
  3. Check Collaboration Status - See if someone else is editing
  4. Check Connection - Verify collaboration is connected

Performance Issues

Symptoms: Slow region loading, laggy interactions, high CPU usage.

Solutions:

  1. Lazy Loading - Regions load as you scroll
  2. Reduce Region Count - Too many regions can slow performance
  3. Check Widget Performance - Some widgets are heavier than others
  4. Browser Optimization - Close other tabs, clear cache, update browser

Best Practices

Layout Organization

  • Group Related Regions - Place related regions together
  • Use Consistent Sizing - Similar regions should be similar sizes
  • Leave Breathing Room - Don't pack regions too tightly
  • Prioritize Important Regions - Place critical regions at top-left

Performance Optimization

  • Limit Region Count - Aim for 10-15 regions maximum
  • Lazy Loading - Regions load as needed
  • Optimize Widgets - Use lightweight widgets when possible
  • Monitor Performance - Check Analytics dashboard

Collaboration Etiquette

  • Communicate Changes - Let team know before major changes
  • Respect Locks - Don't force unlock regions
  • Use Versions - Create versions before major changes
  • Check Presence - Look for presence indicators
